# Introdução
A aula foca na **instalação prática do Argo Rollouts** em um cluster Kubernetes já existente. O professor demonstra o ambiente inicial (usando o Kind), verifica os nodes e namespaces, acessa a documentação oficial do Argo Rollouts e executa os passos de instalação conforme recomendado: criação do namespace e aplicação do manifest YAML com os CRDs.

# Preparação do Ambiente
- **Cluster Kubernetes**:
  - Utilizado o **Kind** (mesmo do módulo anterior).
  - Estrutura simples: 1 control plane + 2 workers.
  - Comando verificado: `kubectl get nodes` → nodes ativos.
- **Namespaces**:
  - Verificado com `kubectl get ns`.
  - Apenas namespaces padrão presentes (nenhuma aplicação ou namespace personalizado ainda).
- **Nenhum recurso extra instalado** → ambiente limpo para a instalação.

# Documentação Oficial
- Site: `argoproj.github.io/rollouts`
  - Mesmo domínio base para todos os produtos Argo (CD, Workflows, Events, Rollouts).
  - Basta alterar o caminho na URL para acessar outro produto.
- Seção de **Installation** consultada diretamente.
- Dois passos principais recomendados:
  1. Criar o namespace `argo-rollouts`.
  2. Aplicar o manifest YAML oficial.

# Instalação Passo a Passo
1. **Criação do Namespace**:
   ```bash
   kubectl create namespace argo-rollouts
   ```
   - Namespace dedicado para isolar os recursos do Argo Rollouts.

2. **Download do Manifest YAML**:
   - URL oficial copiada da documentação.
   - Baixado com:
     ```bash
     wget <URL-do-yaml> -O rollouts.yaml
     ```
   - Arquivo salvo localmente na pasta `infra/` para versionamento.

3. **Análise do YAML**:
   - Contém **Custom Resource Definitions (CRDs)**.
   - Usa `apiVersion: apiextensions.k8s.io/v1` e `kind: CustomResourceDefinition`.
   - Confirma a extensão da API do Kubernetes (conceito visto na aula anterior).

4. **Aplicação do Manifest**:
   ```bash
   kubectl apply -n argo-rollouts -f rollouts.yaml
   ```
   - Instalação concluída com sucesso.
   - CRDs do Argo Rollouts agora disponíveis no cluster.

# Boas Práticas Mencionadas
- **Organização de arquivos**:
  - Sugerido duplicar o comando de criação do namespace no YAML ou em script para reprodutibilidade.
  - Facilita recriação do ambiente caso o cluster seja destruído.
- **Versionamento**:
  - Manter o `rollouts.yaml` no repositório (ex: pasta `infra/`) para IaC.

# Conclusão
A aula cumpre o objetivo de **instalar o Argo Rollouts de forma limpa e organizada** em um cluster Kubernetes. Foram executados com sucesso a criação do namespace dedicado e a aplicação do manifest oficial, que instala os CRDs necessários para habilitar implantações avançadas (como Canary e Blue-Green). O ambiente está agora preparado para a próxima etapa: exploração das APIs, criação de Rollouts e integração com aplicações reais. A instalação foi simples, reforçando que o desafio está no uso estratégico da ferramenta, não na configuração inicial.
