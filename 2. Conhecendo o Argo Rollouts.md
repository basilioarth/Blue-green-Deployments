# Introdução
A aula apresenta o **Argo Rollouts**, uma ferramenta do ecossistema Argo para gerenciar implantações avançadas no Kubernetes. O professor contextualiza o projeto Argo como um guarda-chuva de ferramentas open source voltadas para automação, implantação segura e gerenciamento de aplicações em Kubernetes, destacando seus quatro principais componentes: Argo CD, Argo Workflows, Argo Events e Argo Rollouts.

# Projeto Argo: Visão Geral
O **Project Argo** é composto por ferramentas open source projetadas para aprimorar o fluxo de gerenciamento de aplicações no Kubernetes, promovendo:
- Automação
- Confiabilidade
- Redução de riscos
- Segurança no processo de deployment

### Principais Ferramentas do Argo
- **Argo CD**:
  - Funciona com o conceito de **GitOps**.
  - Usa o repositório Git como fonte única da verdade.
  - Reconcilia automaticamente o estado do cluster com os YAMLs versionados no Git.
  - Comparável a ferramentas como Pulumi, mas específico para Kubernetes.
  - **Não é um CRD**, mas um **controller** instalado no cluster.

- **Argo Workflows**:
  - Motor de orquestração de workflows.
  - Executa tarefas complexas e paralelas em containers.
  - Similar a ferramentas como Go Rules ou Conductor, mas nativo ao Kubernetes.

- **Argo Events**:
  - Gerencia eventos de diversas fontes.
  - Usado para automações reativas, geralmente integrado com workflows.

- **Argo Rollouts** (foco da aula):
  - Ferramenta para implantações progressivas (ex: Canary, Blue-Green).
  - Baseada em **CRD** (Custom Resource Definition).

# Conceito de CRD (Custom Resource Definition)
- O Kubernetes permite **expandir sua API nativa** por meio de CRDs.
- Ferramentas como Argo Rollouts, Workflows e Events são implementadas como CRDs.
- Após a instalação:
  - Novos `kind` e `apiVersion` ficam disponíveis.
  - A API do Kubernetes é estendida para suportar funcionalidades específicas (ex: Canary Deployments).
- **Vantagens**:
  - Integração nativa com o cluster.
  - Possibilidade de criar e compartilhar CRDs personalizados com a comunidade.
- **Instalação**:
  - Geralmente simples (via manifests ou Helm).
  - O desafio está no **uso estratégico**, não na instalação.
  - Considerar consumo de recursos (CPU, memória).

# Argo CD x Outras Ferramentas
- **Argo CD é um controller**, não um CRD.
- Atua em uma camada superior de orquestração.
- Garante sincronia entre Git e cluster.
- Pode usar CRDs internamente para reconciliação, mas não é um em si.
